# Accessibility Audit — Knabberfuchs

## Executive summary
The app is in reasonably good shape on the basics — Material/Flutter auto-semantics cover most labelled fields, switches, and tooltipped buttons — but two systemic themes drive the serious risk. First, **colour contrast in the shared theme is broken at the source**: the muted text token and the brand orange/emerald/amber status colours fall below WCAG 4.5:1 (and sometimes the 3:1 graphical floor), so failures are pervasive rather than local. Second, **dynamic feedback is invisible to screen readers**: every snackbar (including unrecoverable errors) and every loading dialog is silent, and the custom auto-dismiss helper actively defeats Flutter's built-in accommodation that keeps messages on-screen for assistive-tech users. Beyond that, the long tail is a consistent pattern of icon-only `IconButton`s missing tooltips, a few `Row`s that overflow at large text scale, and `GestureDetector`/heading semantics gaps.

## High
- **Light-theme muted text fails 4.5:1 on every surface** — `../fuchsbau/lib/src/theme.dart:33,78-79` — WCAG 1.4.3 — Darken the muted token to ~`#6F6058` for `onSurfaceVariant` (secondary/bodySmall text); keep a separate lighter token for hairline `outline` (only needs 3:1). This fixes the pervasive muted body + section-label text app-wide.
- **"Over" status value text (amber) fails 4.5:1 on the hero card** — `lib/ui/day/day_screen.dart:329-331` — WCAG 1.4.3 — Add an amber on-light text variant (~`#9A6B12`, ≈4.6:1 on white) for the value text; keep `#E0A33B` only for the bar fill. The "X over" headline is currently ~2.22:1.
- **White onPrimary text/icon on orange fill fails contrast** — `../fuchsbau/lib/src/theme.dart:65` — WCAG 1.4.3 / 1.4.11 — White on `#EA7A24` is 2.87:1, failing every `FilledButton`. Either darken primary fills to ~`#C25E12` or set `onPrimary` to dark ink (`#231B16`, ≈6.7:1).
- **Snackbar status/error feedback is never announced to screen readers** — `lib/core/snackbar.dart:10-16` — WCAG 4.1.3 Status Messages — Material snackbars are not live regions, so all success/error/validation feedback is silent. In `showAutoSnackBar`, wrap the content in `Semantics(liveRegion: true, …)` and/or call `SemanticsService.announce(text, Directionality.of(context), assertiveness: Assertiveness.assertive)` for errors.

## Medium
- **Trends line chart has no text alternative** — `lib/ui/trends/trends_screen.dart:371-512` — WCAG 1.1.1 — The fl_chart `LineChart` emits no accessible nodes and per-day status is colour-only. Wrap in `Semantics(label: …)` summarizing range/metric/min/avg/max + days in target, or render a paired off-screen per-day textual list.
- **Trends period prev/next chevron IconButtons have no tooltip** — `lib/ui/trends/trends_screen.dart:154-170` — WCAG 4.1.2 — Add `tooltip: l10n.trendsPreviousPeriod` / `trendsNextPeriod` (same fix `day_screen.dart` already applies to its day chevrons).
- **Clear-search IconButton in food search has no tooltip** — `lib/ui/food/food_search_list.dart:152-159` — WCAG 4.1.2 — Add `tooltip: l10n.actionClear` (or a `clearSearch` key).
- **API-key show/hide toggle IconButton has no tooltip or state** — `lib/ui/settings/settings_screen.dart:588-595` — WCAG 4.1.2 — Add a state-dependent `tooltip: _obscure ? l10n.showKey : l10n.hideKey`, which names the control and conveys current show/hide state. (Flagged under both semantics and forms.)
- **Remove-ingredient IconButton in recipe editor has no tooltip** — `lib/ui/recipes/recipe_edit_screen.dart:228-233` — WCAG 4.1.2 — Add `tooltip: l10n.actionRemove`, ideally with the ingredient name via a `Semantics` label.
- **Portion stepper +/- IconButtons in recipe log sheet have no tooltip** — `lib/ui/recipes/recipe_detail_screen.dart:391-408` — WCAG 4.1.2 — Add `tooltip: l10n.portionDecrease` / `portionIncrease` to both.
- **Portion-count stepper +/- IconButtons in split-meal sheet have no tooltip** — `lib/ui/day/split_meal_sheet.dart:114-125` — WCAG 4.1.2 — Add `tooltip: l10n.portionDecrease` / `portionIncrease` to both.
- **In-range status value text (emerald) fails 4.5:1 on the hero card** — `lib/ui/day/day_screen.dart:329-331` — WCAG 1.4.3 — Use a darker emerald for text-on-light (~`#157A43`, ≈4.6:1 on white); reserve `#1FA85D` for icon/bar.
- **Under status value text (indigo) fails 4.5:1 on the hero-card tint** — `lib/ui/day/day_screen.dart:329-331` — WCAG 1.4.3 — Indigo `#8559D0` drops to 4.14:1 on the `#FDEAD9` tint (the common "to go" state). Darken to ~`#6E45BB`, or compute the status colour against the tint.
- **Amber progress bar fails 3:1 as a non-text UI component** — `lib/ui/day/day_screen.dart:393,418` — WCAG 1.4.11 — Amber fill is 2.22:1 on white. Darken to ~`#C98A1F` (≥3:1) or add a contrasting track/outline so the bar boundary is perceivable.
- **White FAB icon/label on emerald is borderline (3.08:1)** — `../fuchsbau/lib/src/theme.dart:73,146` — WCAG 1.4.11 / 1.4.3 — Passes the icon floor but the app uses `FloatingActionButton.extended` with text labels, so the label fails 4.5:1. Darken the emerald FAB fill to ~`#178A4C`, or use dark ink foreground on extended FABs.
- **Day summary hero Row overflows at large font scale** — `lib/ui/day/day_screen.dart:314-336` — WCAG 1.4.10 / 1.4.4 — kcal number, unit, and status `Text` share a `Row` with only a `Spacer` and no `Flexible`. Wrap the kcal in `Flexible(FittedBox(scaleDown))` and the status in `Flexible(Text(softWrap))`; consider `spaceBetween`.
- **Scale-meal sheet percent/"current → new kcal" Row overflows** — `lib/ui/day/scale_meal_sheet.dart:62-78` — WCAG 1.4.10 / 1.4.4 — Wrap the right `Text` in `Flexible(child: Text(…, overflow: ellipsis))` or use `spaceBetween` with soft wrap.
- **Food form "Nutrition per 100 g" heading + Scan button Row overflows** — `lib/ui/food/food_form_screen.dart:262-284` — WCAG 1.4.10 / 1.4.4 — Wrap the heading in `Flexible(Text(ellipsis))`; at very large scales drop the button to icon-only with a tooltip or move it to its own line.
- **Targets Min/Max numeric fields have no programmatic label** — `lib/ui/settings/targets_screen.dart:280-292` — WCAG 1.3.1 / 4.1.2 / 3.3.2 — `_TargetField` uses `hintText` only (which vanishes when filled, and on weekday rows is a *number*). Add a real `labelText` or wrap in `Semantics(textField: true, label: '$metric $row ${min/max}')`.
- **Loading dialogs convey nothing to screen readers** — `lib/ui/food/recognize_food_flow.dart:111-119` (also `ocr_meal_screen.dart:37-40`) — WCAG 4.1.3 — Bare `CircularProgressIndicator` in a blocking dialog is silent. Add `semanticsLabel: l10n.analysing` and `SemanticsService.announce(...)` on open/close.
- **Settings section headers have no heading semantics** — `lib/ui/settings/settings_screen.dart:719-737` — WCAG 1.3.1 / 2.4.6 — `_SectionHeader` is plain `Text`. Wrap in `Semantics(header: true, …)` so all nine groups support heading navigation.
- **Date-picker AppBar title is a tappable Text with no button role** — `lib/ui/day/day_screen.dart:73-83` — WCAG 1.3.1 / 4.1.2 — A bare `GestureDetector` adds no button semantics. Replace with `InkWell`/`TextButton` or wrap in `Semantics(button: true, label: '<date>', hint: l10n.dayChangeDate)`; mark the dropdown icon decorative. (Also flagged Low under 4.1.2 for the missing hint.)
- **Auto-dismiss SnackBar defeats Flutter's AT keep-open accommodation** — `lib/core/snackbar.dart:10-17` — WCAG 2.2.1 Timing Adjustable — The forced-close `Timer` re-imposes 4s on screen-reader users (Flutter normally keeps snackbars open indefinitely when AT is active). Only arm the timer when `!MediaQuery.of(context).accessibleNavigation`.
- **Critical error feedback only via auto-dismissing SnackBars with no retry** — `lib/ui/settings/settings_screen.dart:236-238,275-276` (also `recognize_food_flow.dart:106`, `recipe_edit_screen.dart:153-154`) — WCAG 2.2.1 — Backup/import/AI errors vanish after 4s with no recall. Give error snackbars a longer duration + `SnackBarAction`, or surface them in a persistent banner/dialog.

## Quick wins
- Fix the **muted text token** (`theme.dart:78`) and the **status-text colours** (over/in-range/under) — a handful of token edits clear the bulk of the High/Medium contrast failures app-wide.
- Add **`liveRegion`/`SemanticsService.announce` in `showAutoSnackBar`** and guard its auto-dismiss `Timer` with `accessibleNavigation` — one helper fixes both the "feedback is silent" and "feedback disappears too fast" themes across every screen.
- Batch-add **`tooltip:` to the icon-only IconButtons** (trends chevrons, two clear-search buttons, show/hide key toggle, remove-ingredient, both portion steppers) — trivial per-line edits closing most 4.1.2 gaps.
- Wrap the **settings section headers** and the **edit-meal sheet title** in `Semantics(header: true)` — two small wraps restoring heading navigation.

## Clean dimension
No findings were raised for **keyboard/focus traps or focus-order** beyond the noted stat-tile ordering, and **no missing-form-label** issues exist outside the Targets Min/Max fields — the rest of the app's `TextField`s rely correctly on `labelText`.
